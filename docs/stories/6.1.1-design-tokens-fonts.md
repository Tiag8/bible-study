## CRITERIOS DE ACEITACAO

### A. Paleta de Cores Parchment

```gherkin
Scenario: Design tokens atualizados com paleta parchment
  Given o arquivo design-tokens.ts
  When verifico os tokens de cor
  Then as cores parchment estao definidas (bg-parchment, bg-cream, bg-ivory, etc.)
  And as cores de texto quentes estao definidas (espresso, walnut, stone, sand)
  And as cores de accent amber estao definidas
  And os tokens anteriores sao substituidos (nao duplicados)
```

**Tarefas Tecnicas**:
- [ ] Substituir COLORS.primary com tokens parchment (accent-amber)
- [ ] Substituir COLORS.neutral com tons quentes
- [ ] Atualizar TAG_COLORS com pasteis quentes
- [ ] Atualizar STATUS_CONFIG com cores pasteis quentes
- [ ] Atualizar BORDERS com cor linen (#EDE8E0)
- [ ] Atualizar SHADOWS com rgba sepia (60, 36, 21)

### B. Fontes Google (Lora + Inter)

```gherkin
Scenario: Fontes Lora e Inter carregadas no layout
  Given o arquivo layout.tsx
  When a pagina carrega
  Then a fonte Lora esta disponivel via CSS variable --font-lora
  And a fonte Inter esta disponivel via CSS variable --font-inter
  And as fontes sao pre-loaded (next/font/google optimization)
```

**Tarefas Tecnicas**:
- [ ] Importar Lora e Inter de `next/font/google` em `layout.tsx`
- [ ] Configurar weights: Lora (400, 500, 600, 700), Inter (300, 400, 500, 600)
- [ ] Adicionar CSS variables `--font-lora` e `--font-inter` no body
- [ ] Manter Geist como fallback (nao remover ainda)
- [ ] Adicionar classes utilitarias `font-lora` e `font-inter` no Tailwind config se necessario

### C. Tokens de Tipografia

```gherkin
Scenario: Tokens de tipografia atualizados
  Given o objeto TYPOGRAPHY em design-tokens.ts
  When verifico os tokens
  Then existe mapping para font-family (serif: lora, sans: inter)
  And a escala tipografica esta definida (h1=32px, h2=24px, body=16px, etc.)
```

**Tarefas Tecnicas**:
- [ ] Adicionar TYPOGRAPHY.families (serif, sans)
- [ ] Atualizar TYPOGRAPHY.sizes se necessario
- [ ] Adicionar line-heights otimizados (1.2 titulos, 1.6 corpo)

### D. Build e Lint

```gherkin
Scenario: Build passa sem erros
  Given as alteracoes em design-tokens.ts e layout.tsx
  When executo npm run build
  Then build completa sem erros
  And npm run lint passa sem warnings novos
```
